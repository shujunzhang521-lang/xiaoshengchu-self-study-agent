# 小升初自学智能体本地网页 MVP 设计

## 背景

用户希望为孩子搭建一个小学升初中的自学体系，并逐步形成一个智能体。孩子是杭州市滨江区学生，2026 年 9 月进入初一。当前目标不是做一个完整教育平台，而是在暑假阶段先搭建一个本地电脑可运行的网页应用，验证孩子是否愿意每天使用，并跑通语文、数学、英语的查漏补缺和初一衔接预习闭环。

孩子画像：

- 小升初阶段，9 月初一开学。
- 语文、数学、英语都需要系统查漏。
- 当前水平偏中等偏上，但不稳定。
- 每天可投入约 90 分钟。
- 第一版以孩子直接使用为主，家长只看简报。

## 目标

第一版 MVP 要让孩子能在本地网页中完成一次完整学习闭环：

1. 看到当天 90 分钟任务包。
2. 按学科完成少量诊断和练习题。
3. 获得即时对错反馈和简短讲解。
4. 为错题选择错因并完成一句话复盘。
5. 在本地保存完成情况、错题标签和知识点状态。
6. 生成一段家长简报，说明当天表现和次日建议。

## 非目标

第一版不做以下内容：

- 不做账号体系和多孩子管理。
- 不做线上部署。
- 不做复杂家长端。
- 不做完整资料上传和 OCR。
- 不追求覆盖杭州全部教材体系。
- 不做大型知识图谱可视化。
- 不依赖真实 AI API 才能运行；第一版可先用内置规则和模拟教练反馈跑通流程。

## 产品形态

应用是一个本地网页 MVP，优先通过浏览器访问本地开发服务器。孩子进入后看到的是一个简洁的学习驾驶舱，而不是复杂后台。

核心体验采用“每日任务 + 知识地图后台 + 教练式反馈”的混合型：

- 孩子前台只看今天该做什么。
- 系统后台按知识点记录掌握状态。
- 答错后给简短讲解和追问式反馈。
- 每天结束生成复盘和家长简报。

## 视觉与体验方向

界面要有明显层次感和亲和力，让孩子愿意主动接触，而不是像成人后台或普通表单。第一版设计方向是“暑假学习驾驶舱”：轻松、清楚、有进度感，但不幼稚。

视觉原则：

- 首页要有强烈的“今天从这里开始”的入口感。
- 学科任务用不同颜色和图标区分，便于孩子快速识别。
- 页面层级分为：今日目标、当前任务、即时反馈、知识状态。
- 每完成一个小任务都要有可见进度反馈。
- 错题反馈不使用大面积红色压迫感，只用温和提示和下一步动作。
- 字体、间距和按钮尺寸要适合孩子独立操作。

建议色彩系统：

- 主背景：`#F6F8FC`，清爽浅灰蓝，减少学习压力。
- 主行动色：`#2563EB`，用于开始、下一题、保存等主要动作。
- 数学色：`#14B8A6`，偏理性和清爽。
- 英语色：`#8B5CF6`，偏记忆和表达。
- 语文色：`#F59E0B`，偏温暖和阅读感。
- 预习色：`#10B981`，表达进入初一的新阶段。
- 提醒色：`#EF4444` 只用于必要错误状态，避免过度刺激。

布局层级：

- 顶部是今日问候、日期和学习能量状态。
- 中间是 90 分钟任务路径，以横向或纵向进度轨道展示。
- 当前任务卡片占据页面视觉中心。
- 右侧或下方展示知识状态和今日小成就。
- 复盘页用更安静的布局，让孩子写一句总结。

签名体验：

- 使用“今日航线”作为任务路径隐喻：数学、英语、语文、初一预习是四个站点。
- 每完成一个站点，站点点亮并出现一句短鼓励。
- 家长简报不做成成绩单，而做成“今日观察”，减少家长和孩子之间的压力传递。

第一版不做复杂动画，但可以加入轻量微交互：

- 任务完成后的勾选反馈。
- 当前学习站点轻微高亮。
- 答对后显示短促正反馈。
- 答错后展开“看提示”和“再试一次”。

## 每日学习结构

默认每天 90 分钟，第一版按以下节奏设计：

- 5 分钟：今日状态选择，孩子选择精神状态和信心。
- 25 分钟：数学查漏。
- 20 分钟：英语查漏。
- 20 分钟：语文查漏。
- 10 分钟：初一衔接预习。
- 10 分钟：今日复盘和家长简报。

实际 MVP 不强制计时到秒，而是用任务进度表达节奏。

## 学科范围

第一版题库用内置样例启动，后续可接入用户上传的教材、试卷、错题和老师要求进行校准。

数学知识点：

- 计算稳定性。
- 分数、小数、百分数。
- 简易方程。
- 几何基础。
- 应用题审题。
- 单位换算。
- 初一预习：有理数、代数式和方程意识。

英语知识点：

- 高频词汇。
- 基础时态。
- 句型转换。
- 阅读理解。
- 简短翻译和表达。
- 初一预习：be 动词、一般现在时、名词单复数、基础句子结构。

语文知识点：

- 字词基础。
- 古诗文积累。
- 阅读理解。
- 概括表达。
- 作文结构。
- 初一预习：文言文意识、现代文阅读答题规范、摘抄积累。

## 题目与作答方式

第一版采用“诊断客观题 + 复盘主观表达”的方式：

- 选择题和填空题用于快速自动批改。
- 短答题用于训练表达，但第一版批改以参考答案和关键词提示为主。
- 错题必须选择错因。
- 每个学习日结束时，孩子写一句自我复盘。

错因分类：

- 概念不清。
- 方法不会。
- 审题错误。
- 计算或拼写错误。
- 表达不完整。
- 粗心。

## AI 教练反馈

第一版先用规则化教练反馈模拟智能体能力，后续再接真实 AI API。

答错后的反馈流程：

1. 告诉孩子这题对应哪个知识点。
2. 用一句话解释关键思路。
3. 让孩子选择错因。
4. 给一个下一步建议。
5. 在复盘中要求孩子用自己的话总结。

反馈语气要像学习教练，不像家长训话：

- 具体。
- 短句。
- 少评判。
- 多提醒下一步动作。

## 本地数据模型

第一版可使用浏览器 localStorage 保存数据，避免引入数据库。

主要数据：

- `dailySessions`：每天的任务完成记录。
- `answers`：题目答案、是否正确、错因。
- `knowledgeStatus`：知识点状态。
- `reflection`：孩子当天一句话复盘。
- `parentBrief`：家长简报文本。

知识点状态：

- `not_started`：未诊断。
- `needs_practice`：需巩固。
- `stable`：基本掌握。

## 页面与组件

首页组件：

- 今日日期。
- 今日状态选择。
- 90 分钟任务包。
- 总进度。
- 开始学习按钮。

学习任务组件：

- 当前学科和任务名称。
- 题目卡片。
- 答案输入或选项。
- 提交按钮。
- 反馈区。
- 错因选择。
- 下一题按钮。

知识状态组件：

- 三科掌握概览。
- 需巩固知识点列表。
- 基本掌握知识点列表。

复盘组件：

- 今日错题摘要。
- 一句话复盘输入框。
- 生成家长简报按钮。

家长简报组件：

- 今日完成情况。
- 不稳定知识点。
- 明日建议。
- 家长鼓励建议。

## 数据流

1. 应用加载内置题库和本地历史记录。
2. 根据当天日期生成任务包。
3. 孩子依次答题。
4. 每次提交后立即批改。
5. 错题进入错因归类。
6. 系统更新知识点状态。
7. 结束时生成今日复盘和家长简报。
8. 数据保存到 localStorage。

## 错误处理

- 如果 localStorage 不可用，应用提示“本次记录无法保存”，但仍允许继续学习。
- 如果孩子未选择错因，错题仍记录为“未归类”，复盘时提醒补充。
- 如果孩子跳过题目，该题不计入正确率，但计入未完成。
- 如果没有历史数据，系统使用默认任务包。

## 验收标准

MVP 完成后应满足：

- 本地网页能正常打开。
- 首页能展示当天任务包。
- 孩子能完成至少一轮数学、英语、语文和初一预习题目。
- 答题后能看到对错和简短反馈。
- 错题能选择错因。
- 今日复盘能保存。
- 家长简报能生成。
- 刷新页面后能看到当天记录仍存在。
- 首页视觉上有清晰层次：今日目标、任务路径、当前行动入口和知识状态可以一眼区分。
- 学科颜色和图标能稳定区分数学、英语、语文和初一预习。
- 页面在桌面浏览器中不显得像普通后台表单，整体应接近孩子愿意打开的学习驾驶舱。
- 主要按钮、题目、反馈和复盘输入区在视觉上有明确优先级。

## 后续迭代

第二阶段可加入：

- 上传教材目录、试卷和错题照片。
- 接入真实 AI API 生成个性化讲解。
- 生成每周报告。
- 增加杭州滨江区和浙江教材衔接知识点。
- 做家长只读视图。
- 部署成可通过浏览器访问的网页链接。
